## 0. 一句话说明

现在"这条视频里哪些话关键、哪些话能拿去验真"全交给不稳的 DeepSeek 一次拍板，它抽风一次整条验真就空转或翻车。
本设计把它改成：**先用确定性的"形式信号"划出关键句骨架，再让 DeepSeek 只在骨架上精修并抽几次取并集，抽完缓存住不重抽。**

---

## 1. 背景与真问题

### 1.1 现状（v0.4.4 实测）
- 同一条视频 BV1jCNe6zEMb 跑 3 遍，`n_claims = 15 / 18 / 13`，`truth_label = suspect / false / true` 三轮翻转。
- 根因不是代码 bug（截断/崩溃已修），是 **DeepSeek 不尊重 temperature=0**，每次抽出的主张集合不同，下游确定性判定（D-S）吃不同输入 → 结论跟着翻。

### 1.2 为什么"不稳"是前提不是 bug（文献支撑）
- Dissecting Non-Determinism (ICLR 2026)、Defeating Nondeterminism (2025) 证明：即使 temperature=0，API 返回仍不稳。根因 = GPU 浮点运算天生不满足结合律（并行线程完成顺序随机，舍入误差累积不同）+ 商用 API 动态批处理（"noisy neighbor"，你的请求和别人随机拼一起跑）。
- **结论**：DeepSeek 不稳是基础设施属性，不是待修的缺陷。正确做法是"以它不稳为前提"做架构兜底，而不是试图让它变稳。

### 1.3 项目既有定论（必须尊重，避免打架）
- `PROJECT_PLAN.md` 第 362 行已记：**"seed+多采样投票对 DeepSeek 无效（复杂判定不尊重 seed、边界内容投票必平局），不采用"**。
- 该结论针对**最终判定投票**。本设计的"自一致性"**只用于抽主张的 N 次并集去重**（不是对真假投票），两者边界清晰，不冲突。

---

## 2. 核心原则（4 条）

1. **以 DeepSeek 不稳为前提**：不指望它每次输出一样，用工程手段抵消方差。
2. **确定性信号优先**：凡能不调 LLM 就确定的（位置、节奏、话术、重复度），先确定，再喂给 LLM 当约束。
3. **抽完即缓存**：一条视频的主张清单第一次抽好就落盘，复查/出报告复用，不重抽（自然消除"同视频三次不一样"）。
4. **结论只由确定性层推导**：最终真假结论只来自 D-S 融合 + MiniCheck（同输入必同输出），绝不采 LLM 自由标签。

## 3. 形式信号骨架（深研：到底怎么算）

> 用户原话："抽关键原话可以和视频形式分析结合，比如总结句、高光、关键词、前10秒、逻辑、AI总结。内容分析和形式分析的成果是可以结合的。"
> 用户的例子是随手举的，本节能把这些直觉系统化、科学化，并落到我们已有的形式线代码上。

### 3.1 关键认知：形式信号 = 确定性显著度（salience）
学术上叫 **salience signals（显著性信号）**。关键句抽取的标准范式是"多信号打分 → 加权融合排序"：
- RankSum (arXiv 2402.5976) 用 主题/语义/关键词/位置 四维排序融合；
- TextRank/LexRank 用图中心度；
- 位置信号来自新闻学"首尾重要"惯例；
- 线索词（"总之/记住/第一第二"）加权。

**最大优点**：这些信号**基本不依赖 LLM、是确定性的**——正好治"不稳"。

### 3.2 可直接复用的已有形式线（form_track.py，零成本）
我们已有 FT0–FT6，且部分是纯函数（FT0/FT6 零 LLM 成本）。骨架可直接消费：

| 信号 | 来源 | 确定性 | 含义 |
|---|---|---|---|
| 节奏/时长分层 | FT0（纯函数） | ✅ 确定 | 句长、停顿节奏，长停顿后句常是重点 |
| 首句强度 / 钩子 | FT1 | 部分确定 | 前 10 秒、开头立论句天然重要 |
| 叙事分段 + 每句目的 | FT2（带时间戳） | 部分确定 | 总结段、转折段、结论段权重高 |
| 人设/信任基石 | FT3 | 确定 | 自报身份、承诺句常是关键主张来源 |
| 修辞话术 span | FT4（22种+规则兜底） | ✅ 确定 | 绝对化/夸张/模糊词所在句 = 高验真价值 |
| 可复制模板骨架 | FT5 | 确定 | 标题公式、步骤骨架 = 结构化主张候选 |
| 情绪曲线（弹幕密度弱代理） | FT6 | weak | 弹幕密集段 = 高光时刻，弱信号参考 |

**复用关键**：FT0/FT1/FT2/FT4 已经对字幕做了带时间戳的句分割和信号标注。抽主张骨架**直接消费这份分割结果**，不另起炉灶、不重复分割。

### 3.3 我们只有字幕时间戳（无原始音频）——伪声学信号可从时间戳算
文献（Whisper 动态字幕）证明字幕时间戳足以推导伪声学特征：
- **语速** = 该句字数 ÷ 句时长（`len(text)/(end-start)`）；语速突增 = 强调。
- **停顿** = 本句 start − 上句 end；长停顿后句 = 重点转折。
- **标点强调** = `！？…` 出现即加权（无需音频）。
- **重复度** = 同一短语跨句出现次数（TF 词频中心度），重复 = 强调/核心。

### 3.4 骨架打分公式（预设权重，#2 后调）
对每条"句子级单元"（来自 FT2 分割）算显著度分：

```
salience(sentence) =
    w_pos   * pos_score        # 位置：前10秒/首句/总结段/结尾段高
  + w_dur   * duration_score    # 时长/停顿：长停顿后、适中句长
  + w_rep   * repetition_score  # 词频中心度 TF-IDF / TextRank
  + w_rhet  * rhetoric_score    # FT4 命中话术数（绝对化/夸张加权更高）
  + w_punct * punctuation_score # ！？… 强调标点
  + w_emo   * emotion_score     # FT6 弹幕密度（weak，低权重）
  + w_hook  * hook_score        # FT1 钩子/首句强度
```

**预设权重（先给，#2 评测集完成后用数据调）**：
```
w_pos=0.22, w_dur=0.12, w_rep=0.18, w_rhet=0.20,
w_punct=0.10, w_emo=0.06, w_hook=0.12
```
（权重和=1.0；rhetoric 给较高权重因为它直接对应"值得验真"；emotion 因 weak 给最低。）

### 3.5 骨架产出
- 按 `salience` 降序取 Top-K（预设 K=12，可调）作为"候选高亮骨架"。
- 每个骨架单元带：时间戳、原话、显著度分、命中的形式信号标签。
- **重叠窗口保险**：分段边界那句在两边各算一次，避免跨句主张（"先做A再做B"被句号断开）漏抽；合并时按文本去重。

## 4. 自一致性抽主张流程（怎么用骨架）

### 4.1 新流程（替换现 extract_key_sentences + extract_claims 的单次调用）

```
输入：字幕时间戳 + clean_text
  │
  ├─[确定性，零/低 LLM]──► 形式信号骨架（§3）→ candidate_skeleton (Top-K 带时间戳原话)
  │
  ├─[约束解码]──► 对 candidate_skeleton 跑 N 次抽取（N 预设=3）：
  │     每次：API 结构化输出(json_object) 强制合法 JSON
  │           提示里固定 candidate_skeleton 作为"必须覆盖的原话范围"
  │           输出 = 该骨架范围内的原子主张清单
  │
  ├─[并集去重]──► 合并 N 次结果，按主张文本归一化后去重 → claims_dedup
  │
  ├─[忠实性自检]──► 每条主张回原文找锚点（§5），标 grounded / ungrounded
  │
  └─[缓存]──► 写 cache/{video_id}.claims.json，复查/出报告直接读，不重抽
```

### 4.2 为什么比"抽一次"稳
- **骨架约束**：LLM 只在已确定的高亮原话里抽，自由发挥空间小 → 方差小。
- **自一致性（Wang 2022）**：N 次抽样取并集，某次抽风漏的别的次补上 → 主张清单更全更稳。注：这是"并集去重"不是"投票"，与 PROJECT_PLAN 禁用的"判定投票"不同。
- **缓存**：固定住一次抽签结果，复查不再漂移。

### 4.3 预设参数（先给值，#2 完成后调）
```
N_SAMPLES       = 3        # 自一致性抽样次数
TOP_K_SKELETON  = 12       # 骨架候选句数
CLAIM_BATCH     = 10       # 每批主张数（防超长截断，沿用 v0.4.4）
MAX_TOKENS      = 4096     # 单次抽取预算（call_llm_json 已支持 2000→4096→8192 递增）
STRUCTURED_OUT  = True     # 启用 response_format=json_object / function calling
CACHE_ENABLED    = True     # 抽完落盘
```

## 5. 忠实性自检（补最大科学漏洞）

> 现状漏洞：上游 LLM 抽的主张从没检查"是否真出自视频原文"。LLM 编一条视频里没有的主张，我们会对着幻影去验真（科研叫 faithfulness gap）。

做法（确定性，不调 LLM）：
- 每条主张 `quote` 与字幕全文做**子串/模糊匹配**（归一化标点、去停用词、最长公共子序列阈值）。
- 命中 → `grounded=True` + 记录锚点时间戳；未命中 → `grounded=False`，标记"待人工/待联网核查"，不直接进验真。
- 这步同时让主张天然**溯源到具体时间戳/原文片段**，呼应 §3.2 的可追溯目标。

## 8. 验收标准（本设计落地后）

- 同一条视频连续 3 次跑，`claim_quotes` 集合**完全一致**（缓存生效，不再漂移）。
- 抽出的主张 100% `grounded=True`（忠实性自检零幻影）。
- 端到端（仅炼真）报告可生成，D-S 结论稳定。
- 不调 LLM 的形式信号骨架可在 <1s 内产出 Top-K 候选（确定性、零成本）。
