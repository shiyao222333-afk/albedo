## 四、广泛研究方法（超出规划已有的部分）

规划 §5.4 / §6.8 已覆盖：MiniCheck、OpenFactCheck、Factool、D-S 融合、JTMS、自洽投票（否决）、SummaC（保真）、TruthfulnessEval、NumTemp、Cross-Document Fact Verification、Acrolinx/Writer。以下为**规划未列、但对炼真有价值**的方法：

### 4.1 事实核查验证方法（逐条核验的"验"怎么做得更准更稳）
- **FactScore**（McGill-NLP, EMNLP 2023）：把长文**拆成原子事实** → 每条独立检索证据 → NLI 判支持/矛盾。思想是"逐原子事实验证 + 检索"，比整体判断稳，可直接套我们的"逐条断言"。
- **FacTool**（Ohio State, OpenFactCheck 组件）：**工具增强**核验——用 代码执行 / 维基 / 搜索 / 知识图谱 工具按领域验证（数值用代码算、事实用搜索）。对我们"数值自洽"很有用。
- **SCALE / ClaimVer**：句子级声明抽取 + 蕴含判定（适合科学/方法类）。
- **VeriScore**：把复杂科学主张**语义分解**成子主张逐条验（适合"行业平均转化率 30%"这类可证伪量化主张）。
- **COVER / SIDE**：上下文感知声明验证 + 自引导检索（先让模型自己想"要查什么"再去检索）。
- **RARR**（Retrieve & Revise）：检索证据后**改写文本使其可溯源**——我们可反向用：报告里每条主张附证据来源。
- **AlignScore**（UCLA, 2023）：双编码器统一事实性打分，**是 MiniCheck 的直接竞品/替代**；AlignScore-base ~355M，本地可跑。值得和 MiniCheck 二选一或并列比对。
- **QAFactEval / TRUE / SaFe**：QA 式事实性、文本蕴含、自事实性微调模型。
- **LogicTree / 形式化验证**：对数学/代码类主张用逻辑树或 Lean 形式证明（对我们领域偏重，暂不需）。

### 4.2 LLM 裁判的稳定性/质量方法（规划否决了自洽投票，以下是替代）
- **过程奖励模型 PRM**（Math-Shepherd/Lee 2024）：不只看最终答案，验证**每一步推理**——比结果投票稳得多。我们可对"逐条核验"用 PRM 思路：每步（抽主张→找证据→判蕴含）都打分。
- **微调小裁判模型**：JudgeLM(11B) / PandaLM(7B) / **Prometheus(7-13B, 带 rubric+参考答案)** / Auto-J(33B)。专用评估模型比提示通用 LLM **一致性高得多**。**Prometheus 的"细粒度 rubric 打分"正好治我们的"连续打分飘"**——把 0-100 换成"按 rubric 逐条给档位"。
- **约束解码（grammar-constrained decoding）**：用 guidance/outlines 把输出**锁死成枚举或 JSON schema**，彻底消除格式漂移和"列表长度随机"（我们 `narrative_segs=0/6/7` 就是没锁）。规划用的 `temperature=0 + 固定枚举` 是弱版，grammar 约束是强版。
- **校准（calibration）**：temperature scaling、verbalized confidence、selective prediction（不确定就 abstain）、从 logprobs 取置信度。我们的 `trust_score` 必须用**人工标定集（评测集）** 校准成真概率才有意义。
- **LLM 辩论（debate）**（Irving 2018 / Du 2023）：两模型辩论、第三方判——减单次偏差但贵；对单人公司性价比低。
- **多模型陪审团 + 分歧检测**：Claude+GPT+Qwen 各判，2/3 一致才采信；同时消"自我偏好"和"单次抖动"。成本较高但最稳。
- **主观逻辑 / 信念函数**：D-S 是一种，另有 opinion pool 做不确定性量化（和 §6.2 的 D-S 同源，可扩展）。
- **检索增强验证（RAG over 证据库）**：把证据库（已入库熔知 + 网络）当成可检索事实基座，验证时查而不是靠 LLM 记忆。

### 4.3 对我们领域（中文 B站 教程/推广、单源、本地 GPU）最该补的 6 件事
1. **真部署 MiniCheck（TT6 实质）** 做逐条核验器 —— 用户决策③，规划早有。
2. **D-S/JTMS 证据驱动聚合** 取代自由 LLM `truth_label`（§6.2）—— 一招同时治稳定+质量+可解释+入库状态脱钩。
3. **约束解码** 锁死所有结构化字段（标签/段数/枚举）—— 灭格式与列表长度漂移。
4. **内容类型路由**：无可核查主张 → `truth = N/A`，不强行判真/疑（治 promo 视频翻盘）。
5. **评测集 + 校准**：30~50 条人工标真/假样本，验证 `trust_score` 到底准不准、把分校准成概率（§6.6）。
6. **Prometheus 式 rubric 打分** 替换质量维（文案/结构/逻辑）的自由 0-100。

### 5.1 我们的差异化（规划 §2.2，不是重造轮子）
竞品多是**端到端英文事实核查器**。炼真独特点：
- **流水线中段定位**：只做认知精炼，对接 Nigredo(采集)→Citrinitas(存储)→Rubedo(变现)，不端到端。
- **平台无关 + 文本类型感知**：中文 B站/小红书/公众号/网页都吃"文字"。
- **多 SOP 并列产出**供 Rubedo 执行。
- **跨源矛盾检测**（未来规模期，竞品空白）。
- **FPF 信任聚合 + 经营视角标注**（变现/引用）。
→ 所以我们**复用 MiniCheck 当核验内核**，不重造 OpenFactCheck；把精力放中文内容 + 业务管线上下文。
