# 炼真「稳定性 + 分析质量」科学方法研究（2026-07-16）

> 缘起：三轮鲁棒性测试暴露 Albedo 判定波动（truth_label 同视频三次 `[suspect, suspect, true]`）。
> 用户追问：「不止是稳定性，分析效果也很重要。固定规则分析效果好吗？有没有别的方式稳定？换换思路，有没有科学方法？」
> 本文回答这三问，并给出改造蓝图。

---

## 〇、一句话回答你的三个问题

1. **固定规则稳，但「稳」不等于「对」** —— 我们的 `trust_score=0.55` 三轮不变，是因为它是纯规则算的；但我们**从没校准过它判得准不准**（没有人工标注集验证）。一个稳定的错误数字仍是错误。
2. **有别的方式稳定** —— 自洽投票（多跑几次取多数）、确定性解码（锁死输出格式）、多模型陪审团、校准修正，都是学界验证过的。
3. **有科学方法，而且直接打中我们两个痛点** —— 事实核查的科学范式是「拆原子主张 → 找证据 → 用确定性核验模型判定」，不是「问 LLM 这视频真不真」。后者在学界已被证明不可靠。

---

## 一、当前三器「判定来源」全貌（复习）

| 数字 | 三轮值 | 怎么算 | 稳不稳 | 去哪 |
|------|--------|--------|:--:|------|
| `trust_score` | 0.55/0.55/0.55 | 纯规则聚合 | ✅ 稳 | 进熔知（存储） |
| `truth_label` | suspect/suspect/**true** | **LLM 一句话整体判断** | ❌ 飘 | 报告标签 + 入库状态 |
| `truth_score` | 40/30/**70** | 同上 LLM | ❌ 飘 | 报告显示 |
| `form_score` | 0.52/0.82/0.76 | LLM 连续打分 | ❌ 飘 | 报告显示 |

**关键：进熔知的是稳定那个（0.55）。飘的是标签 `truth_label`。**

但 `truth_label` 在代码里**直接决定入库状态**（标 `true`→存"真实"，标 `suspect`→存"存疑"）。
也就是同一条视频，今天进熔知是"存疑"、明天重跑可能变"真实"——这是数据一致性真问题。

---

## 二、固定规则分析到底好不好？（诚实评估）

- `trust_score=0.55` 是规则聚合的，稳定 ✅
- 但它聚合的是「未验证证据等级 L1 + 冗余度低 + 自相矛盾无」这类**启发式**，**没有经过人工标注验证其正确性**
- **学界标准**：任何评分器在用于生产前，必须有「校准集」——30~50 条人工标过真/假/好的样本，测评分器与人的一致度（Cohen's kappa > 0.7 才算可用）。我们没做这步。
- **结论**：规则分析**透明、可解释、稳定**，但**有效性未证实**。比 LLM 整体判断好（至少不飘、不瞎编），但离「科学验证过」还差一个校准集。

---

## 三、学界验证过的「稳定」方法（对应我们 4 个根因）

### 1. 自洽投票 Self-Consistency（Wang 2022, ICLR 2023）
- 同一问题跑 N 次（temperature 0.5~0.8 制造多样性），取多数票
- 既稳定又提升准确率——**前提是「有唯一正确答案」的任务**
- ⚠️ 关键限制：开放生成 / 无客观答案的场景，正确不一定聚集，投票只稳不定准。我们「promo 视频判真/疑」正是这种边界场景，所以光投票不够。

### 2. 确定性解码 / 约束生成（grammar-constrained）
- 用 JSON schema + 枚举锁死输出格式，标签只能从 `{真/疑/假}` 选
- 消除「格式方差」，但对「决策方差」只能部分缓解（模型仍选）

### 3. 多模型陪审团（Replacing Judges with Juries, Verga）
- 3 个不同家族模型（如 Claude / GPT / Qwen）各判一次取多数 / 均值
- 同时消除「自我偏好偏差」和「单次方差」
- 成本 ×3，但高价值判定值得

### 4. 校准 Calibration
- 用人工标注集测一致度，拟合修正函数；锁定模型版本
- 这是让「分数有意义」的必做步骤

---

## 四、事实核查的科学范式（质量 + 稳定兼得，打中 truth_label）

### 黄金范式
**原子主张抽取 → 证据检索 → NLI 核验模型判定**
- 这是 FEVER / SciFact 等基准的标准做法

### MiniCheck（EMNLP 2024）—— 就是这个范式的 SOTA 小模型
- 输入 `(文档, 主张)` → 输出 `{支持=1, 不支持=0}`，句子级
- MiniCheck-Flan-T5-Large（7.7 亿参数）达到 GPT-4 水平（74.7% vs 75.3%），成本仅 1/400
- **确定性**（同输入同输出）、**可本地跑**（ollama）、**有 35K NLI 数据微调**
- 我们蓝图里 Layer2 预留的 MiniCheck 就是这个——但它一直没真接，`truth_label` 是另起炉灶的 LLM 整体判断

### 为什么这解决了我们的两个痛点
- **稳定性**：核验是确定性小模型，不飘
- **质量**：任务专用微调 + 证据 grounding，比 LLM 整体判断准（学界原话：「对 judge 看不到的数据做事实性评分，要用基于检索的核查，别用 LLM 整体判断」——这正是我们的场景）
- **透明度**：每条主张有「支撑 / 矛盾 / 无法核实」可溯源

---

## 五、最关键的一招：内容类型路由（解决 promo 视频翻盘）

- 事实核查文献区分「可核查主张（checkworthy）」。教程 / 推广类**几乎没有可核查的硬事实主张**
- **科学做法：按内容类型路由**
  - 有可核查事实 → 走「主张 → 证据 → MiniCheck」核查管线 → 出 `truth_score` + label（都确定性）
  - 无可核查事实（教程 / 推广）→ `truthfulness` 标 **「不适用 / 无可核查主张」**，**不强行判真/疑**；改用「说服 / 操纵强度」（我们 FT4 修辞分析）作为这类内容的质量信号
- 这直接消掉我们测到的翻盘：promo 视频没有可证伪主张，本就不该有 `truth_label` 在飘

## 六、对炼真的改造蓝图（换思路）

### 旧管线（飘）
```
字幕 → [LLM 整体判真/疑] → truth_label（飘）→ 入库状态（飘）
```

### 新管线（稳 + 准）
```
字幕
  → 抽原子主张（LLM 结构化 + 自洽投票 N=5）
  → 过滤「可核查主张」
       ├─ 有可核查 → 检索证据（Nigredo/网络）→ MiniCheck 确定性核验
       │            → 按规则聚合 → truth_score（稳）+ label（确定性推导，不飘）→ 入库状态（稳）
       └─ 无可核查 → 路由到 FT4 修辞/说服分析 → 质量信号（非真值，不飘）
```

**核心思维转变**：
- 不再把「这视频真不真」当成一个 LLM 问题
- LLM 只做它擅长的**抽取**和**检索提问**；**判定**交给专用确定性模型（MiniCheck）
- 不是所有内容都该有真值标签——非事实内容给不同的质量信号

## 七、落地优先级（等你拍板）

| 优先级 | 改造项 | 解决什么 |
|------|------|------|
| **P0** | 入库状态改由确定性 `truth_track.severity` 推，脱钩 volatile 的 `truth_label` | 立刻消除「同视频今天存疑明天真实」的数据不一致 |
| **P1** | 真接 MiniCheck（本地 ollama）做 Layer2 核验；`truth_label` 改为由管线聚合推导 | 质量 + 稳定双升，成本比 GPT-4 低 400 倍 |
| **P1** | 内容类型路由——无可核查主张时 `truth=N/A`，走说服分析 | 消掉 promo 类翻盘 |
| **P2** | claim 抽取 + hook/persona 等仍要 LLM 的部分，上自洽投票（N=5, temp~0.6）+ 约束解码 | 剩余 LLM 部分的稳定性 |
| **P2** | 建 30~50 条人工校准集，测 kappa，校准 `trust_score` | 让「稳定」也变成「准确」 |
| **P3** | 连续分（表达力）改离散档位 + 锚定 rubric | 消除打分聚集（score clustering） |

---

## 八、诚实结论

- 固定规则**比 LLM 整体判断好**（稳、透明、不瞎编），但**不等于已验证正确**——差一个校准集
- 真正的科学升级是**把「判真」从 LLM 整体判断，换成「主张拆解 + 证据 + 确定性核验模型」的管线**，LLM 只做它擅长的抽取和检索提问，判定交给专用确定性模型
- 这同时解决**稳定**（确定模型不飘）和**质量**（证据 grounding + 任务微调），且**成本更低**（MiniCheck 比 GPT-4 便宜 400 倍）
