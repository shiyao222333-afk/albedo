# 炼真（Albedo）形式线（Track B）调研报告

> 调研日期：2026-07-16 ｜ 模式：先研究不定计划（同内容线 / 验真线节奏）
> 目标：把"形式线"研究透——视频的"怎么讲"（钩子/结构/节奏/人设/修辞/可复制骨架），与"内容线"（讲了什么）正交独立，两条线合起来才是一份完整报告。
> 背景：内容线 v0.2.1 已落地（分类→关键句锚定→高光窗口→按类型萃取→保真自检）；验真线 v0.3.0 已落地（逐条断言验真）。形式线是蓝图"优点分析②形式轴"一直埋着、但没落成独立流水线的缺口。

---

## 一、形式线定位：和内容线正交

内容线管"**讲了什么**"（知识/SOP/事实/观点），价值在教程/知识/观点类。
形式线管"**怎么讲的**"（结构/钩子/人设/节奏/修辞），价值在**娱乐/故事类**——这类内容线抽不出"可照搬知识"，价值全在"形式"（你能学它的钩子、结构、人设打法）。

两者正交、互不替代，同一份报告的并列章节：
- 教程类：内容线出 SOP；形式线出"这课是怎么讲清楚的结构模板"（教学技巧可复用）
- 娱乐/故事类：内容线几乎空；形式线出"这个爆款的故事骨架/钩子公式"（整条价值在这）
- 观点/测评类：内容线出论点图/决策表；形式线出"它用什么话术让你信"（修辞桥接到验真）

## 二、形式的可分析维度（来自学术 + 创作圈共识）

| 维度 | 含义 | 业界框架 |
|---|---|---|
| **钩子 Hook** | 前 3–10 秒怎么把人摁住 | 9–10 种钩子类型：curiosity_gap（好奇缺口）/ shocking_claim（惊人之语）/ emotional_peak（情绪峰值）/ number_promise（数字承诺）/ question（提问）/ story_loop（故事钩）/ identity_filter（身份筛选）/ imperative-contrarian（命令+反常识）等 |
| **叙事结构 Arc** | 整条视频的起承转合 | 5 幕：Situation→Desire→Conflict→Change→Result；或 4 拍：Hook/Setup/Escalation/Resolution；短视频 3 幕（≤15s）：痛点→揭示→ verdict |
| **节奏 Pacing** | 语速、停顿、信息密度、模式中断 | 语速（字/分钟）、停顿分布（>200ms 即停顿）、每 3–5 秒一个 pattern interrupt、剪辑加速=情绪爬升信号 |
| **人设 Persona** | 你信的不是内容是人 | 信任基石（专业背书/晒结果/卖惨/自黑）、叙事视角（高高在上/平起平坐/老铁）、人设标签；修辞三角 Ethos/Pathos/Logos |
| **修辞话术 Rhetoric** | 用了哪些说服技巧 | 22–25 种说服技术分类：Loaded Language（带情绪词）/ Appeal to Fear（诉诸恐惧）/ Exaggeration（夸大）/ Appeal to Authority（诉诸权威）/ Repetition（重复）/ Slogans（口号）/ Whataboutism 等 |
| **情绪曲线 Emotional Arc** | 张力—释放的起伏 | 张力:释放 ≈ 1:1；反转落在 60%–85% 处；道德落点（"给家人治病"）才让人想分享 |
| **可复制模板 Template** | 骨架/公式能不能直接套 | 标题公式 + 内容骨架（如"痛点开头→技巧1(反常识)→技巧2(场景)→技巧3(低成本)→总结"）；这是用户"照着抄结构"的直接资产 |
| **结尾转化 CTA** | 怎么收尾引导行动 | rehook（中途再钩）、评论引导、loop 循环设计（结尾接开头自动重播） |

## 三、关键发现：咱们的数据能算什么（可行性矩阵）

**咱们现有数据**（来自馏析中转文件）：`subtitle_lines`（带 [mm:ss] 时间戳，含 start/end）、`highlights`、`danmaku`（带时间戳）、`comments_pinned/top`、`ai_conclusion`、`play_analysis`。
**咱们没有的**：真实留存曲线（B站/YouTube 后台 analytics，爬不到）、视频画面（OCR 已推迟）、音频情感特征。

| 维度 | 可行性 | 数据来源 | 是否需 LLM |
|---|---|---|---|
| 钩子类型 + 首句强度 | ✅ 完全可做 | 前 1–3 条字幕（≈前 10s） | LLM 分类 + 规则首词启发（疑问词/数字/"别/绝不"） |
| 节奏（语速/停顿/停顿分布） | ✅ 完全可做（纯函数） | subtitle_lines 的 start/end 时间戳 | **不需 LLM**，直接计算 |
| 叙事结构分段（带时间戳） | ✅ 完全可做 | 全文字幕 | LLM 分段打标 Hook/Setup/Conflict/Change/Result |
| 人设（信任基石/视角/标签） | ✅ 完全可做 | 全文字幕 + ai_conclusion | LLM 文本分析 |
| 修辞话术检测 | ✅ 完全可做（桥接验真） | 全文字幕 | LLM 抽 span + 技术分类 |
| 可复制模板（骨架+标题公式） | ✅ 完全可做 | 全文字幕 + 标题 | LLM 抽骨架 |
| 情绪曲线（近似） | ⚠️ 弱代理 | 分段情感 + **弹幕密度随时间**作为参与度代理 | LLM 分段情感 + 弹幕统计 |
| 真实留存曲线 / 掉粉点 | ❌ 不可做 | 需平台后台数据 | — |
| 视觉节奏 / 镜头语言 / 剪辑 | ❌ 不可做（OCR 推迟） | 需画面 | — |

**结论**：形式线的"文字可算"核心（钩子/节奏/结构/人设/修辞/模板/情绪代理）v1 全都能做；只有"精确留存曲线"和"视觉节奏"受数据限制，需诚实声明或推迟。

## 四、最关键架构决策：形式线 ↔ 验真线的桥接

**修辞话术是两条线的交集，不能各检一遍。**

- 验真线 v0.3.0 已做 `detect_rhetoric`（绝对化骗局话术 + 水词 + 模糊语），那是"**真假维度**"的红线。
- 形式线的"修辞话术"是"**表达维度**"的特征（22–25 种说服技术）。
- 两者重叠区：绝对化话术、诉诸权威、诉诸恐惧、夸大 —— 既是"形式特征"也是"验真红线"。

**建议方案（强烈）**：形式线负责"**全量修辞技术检测**"（22 种分类，带 span），验真线**消费**形式线产出的修辞结果，不再自己重复检。即：修辞检测单一来源 = 形式线；验真线只做"断言级真假验证"。这样避免重复、且修辞维度更全。

> 这与蓝图"形式轴与真假严格分离"不冲突——分离的是"评价维度"，不是"检测实现"。检测实现合并到一处（形式线），两个维度各自取用。

## 六、落地设计草案（v1 范围，供实现参考）

新建 `core/form_track.py`（与 content_track.py / truth_track.py 并列），纯函数 + LLM 混合：

```
analyze_hook(subtitle_lines)            # 前1-3条→hook_type + first_line_strength + 首词启发
analyze_pacing(subtitle_lines)          # 纯函数：语速(字/分)/停顿列表/长停顿=情绪拍/节奏密度
segment_structure(subtitle_lines)       # LLM：5幕分段，每段带 [mm:ss] 时间戳
analyze_persona(text, ai_conclusion)    # LLM：人设标签/信任基石/叙事视角/ethos-pathos-logos
detect_rhetoric_form(text)              # LLM：22种说服技术分类(带span) ← 桥接验真线
extract_template(title, text)           # LLM：标题公式 + 内容骨架(可复制)
analyze_emotional_arc(text, danmaku)    # LLM分段情感 + 弹幕密度时间轴(参与度弱代理)
_run_form_track(inp, llm_kwargs)        # 串联 + 聚合为 form_track dict，全程降级不抛
```

数据模型 `RefinedKnowledgeObject` 增：`form_track: dict`（含 hook/pacing/structure/persona/rhetoric/template/arc）。
报告 `core/report.py` 增 `_render_form_track(o)`：并列"🎬 形式分析"章节（在结论卡、内容线、验真线之后）。

**运行范围**：形式线对**所有内容类型**都跑（教程类出"教学结构模板"，娱乐类出"故事骨架"，价值不同但都正交有用）。

## 七、诚实边界（v1 必须声明）

1. **精确留存曲线**：爬不到 B站后台数据 → 用"弹幕密度时间轴"做参与度弱代理，报告明确标"⚠️ 非真实留存，仅弹幕参与度近似"。
2. **视觉节奏/镜头语言**：需画面 → OCR 已推迟 → 报告标"🎥 画面节奏未分析"（接验真线 OCR 路线图）。
3. **情绪曲线精度**：无音频/画面 → 文本情感 + 弹幕代理，标注"近似"。
4. **不自动改写脚本**：抽骨架/公式即可，"照着写"是凝华(Rubedo)/总指挥部(OpusMagnum)的活，炼真只交付"可复制模板"。

## 八、待用户拍板的决策点

1. **v1 范围**：一次做全"文字可算"核心（钩子/节奏/结构/人设/修辞桥接/模板/情绪代理），还是先挑最有用的一两个？
   → 建议：一次做核心（纯函数节奏零成本，其余 LLM 调用复用现有桩模式）。
2. **形式线跑全部类型 vs 只娱乐/故事**？
   → 建议：全部类型（正交、教程类也有"教学结构"价值）。
3. **修辞检测合并到形式线、验真线消费（单一来源）？**
   → 建议：是（避免重复、维度更全）。这意味着 v0.3.0 的 `detect_rhetoric` 要反向被形式线 `detect_rhetoric_form` 吸收——需小幅重构验真线。
4. **可复制模板输出粒度**：标题公式 + 骨架（带时间戳）即可，还是也要"填空式模板"？
   → 建议：标题公式 + 骨架 + 每节"这节在干嘛"一句话。
5. **情绪曲线用弹幕密度代理还是先不做**？
   → 建议：用弹幕密度代理 + 诚实声明（成本低、有信号）。
