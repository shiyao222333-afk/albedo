## 2. 核心发现

### 2.1 总结的三大科学范式

| 范式 | 做法 | 幻觉风险 | 可读性 | 适合场景 |
|------|------|:--------:|:------:|---------|
| **抽取式 Extractive** | 挑原文最重要句子拼起来，不改写 | 近零（输出永远是原文子集） | 较低（可能不连贯） | 法律/合规/精确措辞要求 |
| **生成式 Abstractive** | 重新组织语言生成新句 | 0.7%–14%（取决于模型/任务） | 高（自然流畅） | 报告/综述/面向读者 |
| **混合 Hybrid** | 先抽取关键句/事实 → 再生成连贯摘要 | 低 | 高 | 技术文档/学术/我们这种"保真+可读"场景 |

**关键结论**：用户的两条硬要求——「不丢关键信息」+「措辞可不同」——恰好指向**混合范式**：
- 抽取兜底 = 不丢关键信息（原文在，就不会凭空消失）
- 生成只做语言润色 = 措辞可变，但内容被抽取层锁死

学术佐证：大模型时代 LLM 通过提示工程模糊了抽取/生成边界——"先摘录 5 个最重要句子，再基于这些句子写连贯摘要"（思维链引导），或约束解码强制引用原文句子编号。无需专门训练。

### 2.2 长文本（字幕）怎么处理不丢信息

字幕动辄几千到上万字，一次性塞给模型会翻车，有两个物理缺陷：
- **Lost in the Middle（中间迷失）**：模型对文档开头/结尾注意力最强，中间核心信息极易被忽略或幻觉化。
- **注意力稀释**：文本越长，模型越倾向抓"高频套话"，失去深度。

三大工程模式：

1. **Map-Reduce（分治）**：切块 → 每块独立摘要（map）→ 合并（reduce）。可并行，能扩到任意长度。
   - 弱点：块边界可能切断上下文；跨块信息可能丢；reduce 可能丢细节。
   - 对策：每块摘要要"自包含"；reduce 递归直到收敛；或改用分层。

2. **分层摘要（Hierarchical）**：按文档结构（章节/段落）逐级汇总，天然产出多分辨率（一句话总览 / 一段节选 / 详细逐节）。
   - 优势：保留原文组织结构，用户可从概览下钻到细节。

3. **迭代精炼（Iterative Refinement）**：先粗摘 → 逐块对照原文补遗漏/纠错 → 重复直到稳定。
   - 质量最高（自我修正遗漏与错误），但慢、不可并行。
   - 特别适合"不能丢关键信息"的硬要求。

**与用户第 4 点的契合**：用户要求高光窗口按"前后 N 条字幕"而非秒数 —— 这正好是 Map-Reduce 的 **chunk 边界 = 语义窗口（字幕条数）**，比按固定秒数切更语义对齐（一条字幕可能跨多秒，一个知识点可能跨多条字幕）。

### 2.3 忠实度：怎么保证"不丢关键信息、不编造"

**幻觉的两类**（生成式摘要最致命弱点）：
- 内在幻觉：摘要与原文**矛盾**（如把"799美元"说成"999美元"）。
- 外在幻觉：加了原文没提但看似合理的信息（如从"苹果发新手机"推断"库克主持发布会"）。

**自动评估指标（专门检测幻觉，ROUGE 做不到）**：
- **FactCC**：训练 BERT 判断摘要是否与原文事实一致（替换/删/反义构造负例）。
- **QuestEval**：把一致性转成问答——从摘要生成问题，在原文找答案，算匹配率。
- **SummaC**：原文/摘要都切句 → 算句子对 NLI 蕴含矩阵 → 聚合为 0–1 文档级分数（每句摘要至少被一句原文蕴含才过关）。**零样本可用**，与人工判断相关系数最高。
- **NLI 一致性**：每句摘要判 蕴含/矛盾/中立；矛盾=幻觉。

**提升忠实度的技术**：抽取优先 → 约束生成（引用原文编号）→ 可控生成（前缀提示引导忠实）→ 分块（减少注意力稀释）。

**对本项目的直接落地**：
1. 混合范式（抽取层兜底）从架构上防丢。
2. 每条摘要句**强制带字幕时间戳/来源**（溯源）——既满足"不丢关键信息可核查"，又呼应蓝图溯源要求。
3. 可用 SummaC 类 NLI 做**自动质检**：同输入跑两次，若双向蕴含则一致；若某句无原文支撑则告警。

### 2.4 一致性：用户第 5 点"措辞可不同、内容一致"

用户定义的是**语义一致性**，不是字面一致性。科学实现：

**评估方法**：
- **NLI 双向蕴含**：摘要 A 蕴含 B 且 B 蕴含 A → 语义等价（措辞不同但内容一致）。
- **BERTScore**：基于上下文嵌入的语义相似度，捕捉同义改写（ROUGE 会误判同义替换为"不一致"，BERTScore 不会）。
- **SummaC**：两者都对照原文，看对原文忠实度是否都高。

**工程实现（确定性铁律，呼应前几轮调研）**：
- 固定码本（固定输出框架：优点/可照搬步骤/避坑/核心洞察）→ 内容维度被锁死
- 约束解码 / JSON Schema 枚举 → 模型"不能"违规
- temperature = 0 → 关掉随机性
- 单维度评分 → 避免复合判断漂移
- 分块一致性：每步用相同 prompt 模板

**洞察**：用户第 5 点（内容一致措辞可变）与第 6 点（不丢关键信息）是**同一个工程解的两面**——固定框架 + 抽取兜底 + 溯源 + NLI 质检，一次性同时满足。

### 2.5 结构化总结框架：金字塔原理 + MECE

- **金字塔原理（Minto）**：结论先行 → 支撑论点（MECE：相互独立、完全穷尽）→ 论据。读者先看到核心结论。
- **MECE**：分组不重叠（Mutually Exclusive）+ 不遗漏（Collectively Exhaustive）→ 正好对应"不丢关键信息"（完全穷尽）且不重复。
- **SCQA 叙事**：情境-冲突-问题-答案，用于开场抓注意力。

**对本项目的落地**：炼真"鉴定报告"用金字塔结构——
1. 核心结论先行（可信度 + 一句话定性，如"可信度中，含 3 个可照搬步骤 + 2 个避坑点"）
2. 按 MECE 分组：优点 / 可照搬步骤 / 避坑点 / 核心洞察
3. 每条带字幕时间戳（溯源）

### 2.6 视频字幕总结的特殊性 + 竞品做法

**特殊性**：字幕口语化、冗余、碎片化（"呃""然后""对吧"）→ 需先"口语→书面"归一（去口水话、合并碎句）再提炼，否则噪声进模型。

**竞品做法（2025–2026 实测）**：
- **YouTube Summarizer API**：结构化 JSON（summary + key_points + standout_quote 原文金句），长视频切块 map-reduce，分句边界切。
- **youtube-digest**：三层字幕回退（官方→自动→Whisper）+ 语义重叠分块（重叠量可配）+ 多格式输出（单页摘要 / 优先排序重点 / 教学 Skill.md）。
- **TicNote**：**先定"目的"再定义"价值单元"**——学习：概念/定义/例子/为什么；工作：决策/约束/风险/步骤；内容：钩子/结构/金句/论据。这直接回答用户第 1 点。
- **百度 AI 大模型总结**：JSON Schema 输出 + temperature=0.3 保稳定 + ROUGE-L/BERTScore 检覆盖率 + 多轮反馈修正。

**竞品留白（我们的护城河，更新）**：
1. 没人把"抽取兜底 + 忠实度 NLI 质检 + 固定码本 + 溯源 + 确定性输出"组合进同一条视频流水线；
2. 组合"字幕(核心) + 高光 + 弹幕 + 评论"做内容萃取的极少；
3. 从**别人教程视频抽可照搬 SOP**（非录自己屏）是空白；
4. 把"不丢关键信息"当作可量化质检指标（SummaC 分数）的，几乎没有。

### 2.7 视频内容分类：不能只二分（用户第 1 点）

科学分类法不止"教程/观点"两类：

- **按目的（videomaker）**：inform 资讯 / instruct 教学 / persuade 说服 / preserve 留存 / entertain 娱乐。
- **学习视频三角（scottscoffeehouse）**：Inspirational 激励 / Informational 告知 / Instructional 教学演示。
- **YouTube 8 格式（littlemonster）**：Listicle 清单 / Explainer 讲解 / Commentary 评论 / Interview 访谈 / Music Video / Challenge 挑战 / Reaction 反应 / Narrative 叙事。
- **叙事类型（抖音硕士研究）**：展示类 / 介绍类 / 推荐类 / 攻略类。
- **内容策略 6 类（blumars）**：教育 / 品牌 / 说服 / 行动 / 信任 / 情感故事。

**结论**：视频类型远多于两类。**正确做法不是预设固定几类，而是按"用户想从这条视频拿到什么价值"动态路由**（TicNote 的"目的→价值单元"法最实用）：
- 学习/涨知识 → 抽概念、定义、例子、"为什么"
- 工作/能照搬 → 抽决策、约束、风险、步骤（SOP）
- 内容创作/拆解 → 抽钩子、结构、金句、论据
- 观点/立场 → 抽论点、论据、反驳

→ 内容线"类型分流"应是**价值导向的动态路由**，自动分流可行（让模型先判"这条视频主要提供哪类价值"），但分类维度要比"教程/观点"细，且按价值单元决定抽什么。
