# ADR-003: 炼真产出形态与内容标注扩展（双输出 / 网页输入 / 变现标注 / 引用标记）

## Status
Accepted（2026-07-09）

## Context
用户确认 A（将新竞品补入文档）后，进一步提出 4 项产品形态 / 标注要求：

1. **双输出**：炼真总结的文档除了导入熔知，还要能直接给人看。
2. **网页输入（未来）**：未来应支持读取网页（不止视频/社媒/文档）。
3. **变现标注**：变现仍是关键——内容与变现直接相关应标注出来。
4. **引用标记**：文档若引用别的信息（推荐书/网址），应标记出来。

## Decision

- **双输出（同源渲染）**：炼真产出单一事实源 `RefinedKnowledgeObject`（结构化，交熔知入库）+ 由其渲染的**人类可读鉴定报告**（Markdown/HTML，人直接看、直接决策）。报告是对象的渲染，非另写一份，避免双源失真。v0.1.0 最小报告由 UI 直出，v0.2.0 完整报告含全维度 + 引用 + 变现标注。
- **网页输入（未来）**：`text_type` 增加 `webpage`；网页抓取由适配器（Nigredo 系或独立 fetch 适配器）完成，炼真仍只吃文字、不自己抓 URL。与熔知 v1.5.0「网页 URL 直接摄入」的边界：熔知 = 不炼直接存，炼真 = 抓来炼了再存；五器合体时由 OpusMagnum 编排决定走哪条，本期仅预留 `text_type`。
- **变现标注**：新增 `monetization{related, category, note}` 字段——标记内容本身「在卖什么」（卖课 / 带货 / 工具付费）。与已移出的「业务线适配评分」严格区分：**前者是内容客观属性（留炼真）**，后者是「对你生意适不适用」的评分（已移出归 Rubedo/OpusMagnum）。报告醒目展示。
- **引用标记**：新增 `references[{type, value, context}]` 字段——抽取内容里提到的书 / 网址 / 资料并结构化，方便顺藤摸瓜与下游可信度判断。

## Consequences

- **更易**：主理人直接消费成果；经营视角（变现）标注贴合一人公司；溯源增强。
- **更难**：需维护报告渲染模板；引用 / 变现抽取增加 LLM 输出字段与校验。
- **依赖**：网页输入依赖适配器（Nigredo 范围）；双输出不影响熔知接口（仍交结构化对象）。
