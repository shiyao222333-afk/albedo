- 从没做一步："**计划承诺了什么 → 实际交付了什么 → 差哪条**"。所以 TT6 标 ✅ 但实际只是接口壳、§6.2 标 v0.3.0 但没实现，都没被拦截。

## 四、结合调查研究的改造方案

基于两篇研究（`STABILITY-QUALITY` / `METHODS-AUDIT`）与本次审计，方案分三档。核心思路：**LLM 只做"抽取+提问"，"判真"交给确定性证据链**。

### P0（立刻做，成本最低、风险最高）
1. **入库状态脱钩 volatile label**
   - 改 `flows/refine.py:206`：`status` 改由 `truth_track.severity`（确定性）推导；`truthfulness.label` 只作展示参考，**不再决断"真实"**。
   - 效果：立刻消除"同视频今天存疑明天真实"的数据不一致（L4 暴露的头号 bug）。
2. **真部署 MiniCheck（TT6 实质）**——你拍板③
   - 本地 GPU 跑（RTX3080 + Ollama），对 `check_worthy & public & factual` 的断言逐条 `supported/contradicted`；`accuracy` 不再全标 `unverified`。
   - 研究佐证：MiniCheck-Flan-T5-Large（7.7亿）达 GPT-4 级（74.7% vs 75.3%），成本 1/400，确定性不飘。
3. **§6.2 判定方法论重做（治本）**
   - 复用 `D:\albedo-old`：`claim_extractor`（已有 TT1 抽取）→ `verify`（MiniCheck P0-2 核验）→ `ds_fusion.py` / `tms.py` 聚合 → 推导 `truth_label` / `epistemic_status`。
   - 从此 label 由证据链推导，不再自由 LLM —— **同时治稳定 + 质量 + 可解释 + 入库脱钩**。

### P1（稳定剩余 LLM 部分 + 治 promo 翻盘）
4. **内容类型路由**：无可核查主张（教程/推广类）→ `truth = N/A`，走 FT4 修辞/说服分析；不强行判真/疑（直接消掉 promo 视频翻盘）。
5. **约束解码**：所有 LLM 结构化输出锁死 schema/枚举（灭 `narrative_segs=0/6/7`、`hook_type` 漂移）。
6. **G1 真相错觉防御**已落地，保留。

### P2（验证"稳但不准"）
7. **评测集 + 校准**（原排 v0.5.0，建议提前埋种子）：30~50 条你亲自标真/假的样本，验证 `trust_score` 到底准不准、用温度缩放校准成真概率。
8. **研究补充的科学方法**（规划未列，按需取）：
   - `AlignScore`（UCLA）— MiniCheck 直接替代/并列 A/B；
   - `FactScore` / `FacTool` — 原子事实+工具核验，治数值自洽；
   - **PRM 过程奖励模型** — 对"抽主张→找证据→判蕴含"每步打分，比结果投票稳；
   - **校准**（temp scaling / verbalized confidence / selective prediction 不确定就 abstain）。
