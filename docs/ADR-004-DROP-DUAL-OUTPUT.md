# ADR-004: 取消「双输出」对外维护 —— 以人类可读报告为主交付物

## Status
Accepted（2026-07-09）— 部分取代 ADR-003 的「双输出（同源渲染）」决策

## Context
ADR-003 提出「双输出：结构化 `RefinedKnowledgeObject` + 人类可读鉴定报告」。用户反馈：用不着双输出，人能看的就够了。即：对外只交付人类可读鉴定报告，不必把"维护两套对外交付物"当成需求。

## Decision
- 炼真**对外主交付物 = 人类可读鉴定报告（Markdown）**。主理人直接看、直接决策。
- 结构化 `RefinedKnowledgeObject` 仍保留为**内部表示**：① LLM 本就以 JSON 产出，天然存在；② 未来交熔知入库需要。但它不对外强调、不单独维护为"第二交付物"。
- 故 ADR-003 的「双输出（同源渲染）」决策撤销——改为「单主交付物（报告）+ 内部 JSON 副产品（零额外维护）」。

## Consequences
- **更易**：用户只面对一份能读懂的报告，无认知负担。
- **更难（无新增）**：JSON 仍由 LLM 生成，但无额外"渲染维护"成本——报告即由同一份 JSON 渲染，属同一步。
- **依赖**：未来熔知入库时仍从内部 JSON 取结构化字段，不受影响。

## 影响的文档
- `BLUEPRINT.md`：准则 #6 改为「以人能直接看的鉴定报告为关键交付物」。
- `FLOWCHART.md`：接口边界「双输出」段改为「以人能看的报告为交付物」。
- `PROJECT_PLAN.md`：T12 由「报告渲染（双输出之一）」改为「鉴定报告产出（主交付物）」；数据模型 `report` 注「主交付物」。
