# 炼真（Albedo）项目蓝图

> 本文件是炼真项目的"宪法"。只说清楚这个项目**做什么、为什么做**，不说具体怎么做（怎么做在流程框图和计划里）。
> 最后更新：2026-07-17

---

## 一、这个项目做什么（核心定位）

炼真是"一人公司知识流水线"里专门负责**去伪存真、提炼价值**的环节。

简单说，别人（或工具）从网上把教程、经验、卖课视频采集回来，变成一大段文字；炼真接手这段文字，做四件大事：

1. **分辨真假** —— 这条内容可信吗？是真的经验，还是卖课的夸大/谎言？
2. **提炼优点** —— 它做对了什么？有哪些值得我模仿的地方？
3. **整理成能照做的步骤** —— 把杂乱的口播，变成我可以直接照搬的标准动作。
4. **记清楚来源** —— 这段经验来自哪个视频、哪一分哪一秒，方便回头查证。

做完这四件，炼真把"鉴定过、提炼过"的干净内容，交给下一个环节存进知识库。

**为什么需要它**：网上真教程和卖课谎言混在一起，不这道关，脏东西会原样灌进知识库，越积越脏。炼真就是流水线上那道"质检 + 提纯"的关卡。

## 二、炼真具体做哪些事（核心能力，大白话）

### 现在就做（MVP：分 v0.1.0 / v0.2.0 两版落地）

**① 质量评估 —— 给内容"多维鉴定"**
不只判真假，而是分成多个维度分别打分：**真实性**（内容真假，核心）、**文案质量**（表达是否清晰吸引人）、**结构**（组织是否有条理）、**逻辑**（论证是否严密）。一条视频可以"内容假但文案漂亮、逻辑严谨"——所以不能用一个总分盖棺，要分维度看。
- v0.1.0 先做「真实性」这一个维度（验真假核心闭环）；其余维度在 v0.2.0 补全。
- **验真假方法**：大模型判断 + **统计学手段**（跨源共识频率、数值自洽、离群检测）双管齐下。MVP 单源先做轻量"数值自洽"，规模期做"跨源共识"。

**② 优点分析 —— 把"好在哪"挖出来**
从一条经验里，提取值得模仿、可迁移的部分。具体拆成 8 个小件（分"内容轴"和"形式轴"两路）：

内容轴（讲的内容本身好不好）：
- **核心洞察**：一句话点出他做对的本质
- **可复用步骤**：拆成能直接照搬的动作序列（对应你"多 SOP 并列"的打法）
- **差异化亮点**：相比普遍做法，他独特在哪
- **适用场景**：这套打法适合什么阶段、什么生意（通用描述，如阶段 / 行业 / 技能门槛；不做单条业务线适配评分）
- **陷阱预警**：他踩过的坑、别人容易误用之处
- **迁移成本**：照搬这套需要什么前置条件

形式轴（表达本身优不优秀，与"真假"严格分离——讲得精彩不代表可信）：
- **表达形式质量**：他讲得清不清楚、结构好不好、用了什么可学的技巧（学"怎么讲"，不评价"讲的真假"）
- **格式可复用**：这套内容的骨架 / 模板能不能直接套用（如小红书笔记结构、B站教程结构），对你"照着抄结构"直接有用

**③ 结构化提炼 —— 乱文字变标准方法**
把一段东拉西扯的口播，整理成有结构、能照做的标准步骤（目的 → 前置条件 → 步骤 → 注意点 → 完成标志）。
  - 📚 **引用标记**：内容里提到的书、网址、资料、其它出处，单独拎出来标好（类型 + 原文 + 出处上下文），方便你顺藤摸瓜，也方便下游判断可信度。

**④ 来源溯源 —— 记得从哪来**
记录每条知识来自哪个视频、哪个 UP主、哪一分哪一秒，将来要复核随时能找到原处。

**⑤ 内容净化（顺手做）**
- **自动纠错**：把语音转文字产生的错别字改对
- **去广告**：去掉视频里夹带的推广废话
- **翻译**：外文内容翻成中文
- 📌 **字幕与结构化文案处理不同**：同样是"文字"，口语字幕（零碎、有语气词、无段落）和条理清晰的文案（小红书笔记、文章）处理方式不一样——炼真用一个「文本类型」标记区分，字幕走 ASR 清洗 + 更多结构整理，结构化文案直接提炼。炼真不绑平台，但认文本类型。

> 📥 **输入来源与平台信号**：炼真只吃「文字」，但来源会越来越多——视频字幕、微信/小红书文案、未来 PPT/Excel 文档文字（各格式抽取归馏析的"一对象一适配器"）。此外，平台还会带来**内容之外的信号**（如 B站 播放量 / 受众画像 / 点赞收藏评论），由馏析把各平台五花八门的元数据**归一化成统一信号包**（互动热度 / 受众契合 / 口碑情感），炼真只吃归一化信号、不碰原始平台字段——保持平台无关。

**⑥ 内容摘要（最底层基础层）**
在"评价好不好、真不真"之前，先有一层最基础的——把这篇内容**到底讲了什么**，用大白话概括出来（一句话总结 + 几个要点 + 关键主张）。它**只说"讲什么"，不评级、不夸好、不判真假**，是后面所有分析（优点、结构、报告）共用的"压缩包"底座，也作为鉴定报告最开头的"这篇讲了啥"。

### 将来会做（规模更大时再上）

- **跨来源矛盾检测**：拿多个教程互相比对，看是否说法冲突（现在先单条鉴定，等积累够了再做跨条比对）
- **时效判定**：这条经验是不是过时的旧打法
- **冲突仲裁**：多个来源打架时，给"该信谁"的建议
- **去重 / 近重复检测**：同一条视频被采两次、或不同 UP 讲同一件事，炼真前先去重，避免重复炼、白花钱（优先项 ★）
- **超长内容切块**：2 小时课 vs 30 秒技巧长度差百倍，太长要先切块再评，否则 LLM 丢上下文（优先项 ★）
- **可解释到根因**：标"可疑"必须能点开看"因为第 X 条证据不足 / 数值前后矛盾"，否则你不敢信（优先项 ★）
- **新颖度 + 上手门槛维度**：这条是老生常谈还是新打法（新颖度）、做这事要多少工具/技能/钱（上手门槛）——对你挑"值不值得做"特别有用（优先项 ★）
- **变现 ≠ 差内容护栏**：卖课的不一定假、免费的不一定真；变现标注不能和"可疑"画等号，报告里分开呈现（优先项 ★）
- **多平台入口扩展**：现在只接 B站视频，将来接小红书文案、文章、网页等多源（平台归馏析适配器，炼真只吃统一文字 + 文本类型标记，分析方法不变）
- **增强内容 + 形式分析以便学习模仿**：把"好在哪 / 怎么做到的"讲得更透、更结构化、可直接照抄照学，让分析报告本身成为可学习的教材

> 注：「4 条业务线适配度评估」已移出炼真——归凝华（Rubedo）SOP 建立 / 总指挥部（OpusMagnum）编排。炼真只产出通用 SOP 与适用场景标签，不做生意适配评分。
> 注（续）：以上仅列**优先项 ★**。完整多步演进路线（v0.1→v1.1 共 8 版本 / 7 阶段）见 `PROJECT_PLAN.md` **第三节**；22 项研究课题逐项落入目标版本（已在计划中排期）见 `PROJECT_PLAN.md` **第六节（研究课题与版本映射）**。

## 三、在流水线中的位置

```
采集（馏析）  →  【炼真：本项目】  →  存储（熔知）  →  变现（凝华）
 把内容拿回来      验真假、提优点        把干净的存好      拿去赚钱
```

- **上游（馏析）**把各平台的视频字幕 / 社媒文案 / 未来文档（当前以 B站 为主）变成文字（并标文本类型、附平台归一化信号），交给炼真。
- **下游（熔知）**接炼真产出的干净内容，存进知识库。
- **炼真专心做中间这一步**：把内容"炼"干净、"炼"出价值。

## 四、当前重心（第一步先做这个）

做出**第一个能用的版本（最小闭环）**：

> 丢一条从各平台采集来的经验文字（字幕或结构化文案均可，当前以 B站 为主）进来 → 炼真产出【多维鉴定报告：真实性评分 + 文案/结构/逻辑维度 + 核心优点 + 能照着做的标准步骤 + 来自哪个视频哪一段】→ 先存在本地让我能看到，将来直接交给熔知入库。

也就是：v0.1.0 先跑通「内容净化（按文本类型）+ 质量评估·真实性维度」核心闭环；v0.2.0 补全「优点分析 + 结构化提炼 + 来源溯源 + 其余质量维度 + 报告渲染」为完整多维鉴定报告；**v0.2.1 进一步针对字幕类输入新增「内容线」——先判断视频类型（教程/测评/观点/科普/娱乐/叙事），再按类型抽你要的东西（可照搬 SOP / 决策表 / 论点图 / 概念卡），每句话都钉死在字幕哪一句、自动查有没有瞎编**。验真（真/假/可疑）的深化研究（§6.2 判定方法论重做，证据链取代自由 LLM）本轮启动，`assess.py` 作为 MVP 占位暂不动（AI 设计决策，非用户指令）。
**v0.3.0 落实验真环节（逐条断言验真）**——把视频里的主张一条条揪出来分别验真假：先防 AI 自己瞎编断言（Layer0.5 拿字幕原文核对、无支撑的直接丢弃），再做不联网快筛（绝对化话术识别 / 视频自相矛盾 / 时效标记 / 事实vs观点·个人vs公开分类），并做联网深验（MiniCheck 本地部署，你拍板③）；结果逐条落进报告，文档级信任分与入库状态（真假 / 个人公开 / 可信度）直送熔知。OCR 跨模态与 UP 主跨视频信用累积本期只定义字段、列入路线图。

**v0.4.0 补齐形式线（Track B：怎么讲的）**——在"内容线（讲什么）+ 验真线（真不真）"之外，把"形式线（怎么讲）"系统解剖：钩子 / 叙事结构 / 节奏 / 人设 / 修辞话术 / 可复制模板 / 情绪曲线。它与优点分析的"形式轴"**互补不冲突**：优点分析的形式轴是"值不值得学"的浅层评价（presentation_craft / format_reusable），形式线是"怎么讲"的客观结构拆解（不评好坏、只拆骨架，供你照着抄）；修辞话术在形式线做**单一来源**，验真线直接 import 消费、不重复维护。形式线通过 **G1 反向桥**与验真联动——"讲得精彩 + 证据未验证"会下调验真信任分，专防"包装好 = 可信"的错觉；报告升级为「三轴总览：干货度 / 可信度 / 表达力」。OCR 跨模态、UP 主跨视频人设累积本期只定义字段、列入路线图。

## 五、做事的准则（5 条）

1. **专心做中间这步** —— 只做"验真与提炼"，上游采集、下游存储都不碰，把这一件事做到位。
2. **可信优先于全面** —— 宁可漏掉一些，也绝不把假的、夸大的放进来。"少而准"比"多而杂"重要。
3. **跨源印证，不替你拍板** —— 多个来源说法一致才更可信；单个来源存疑就标"还没印证"，把判断权留给你。
4. **提炼必须可操作** —— 优点分析不能只讲大道理，必须拆成能直接照做的动作。
5. **独立做，但对齐接口** —— 现在单独做，产出格式对齐熔知的入库要求，将来五器合体时能直接接上。
6. **以人能直接看的鉴定报告为关键交付物** —— 炼真产出一份说人话的鉴定报告（Markdown，你直接看、直接判断要不要用）。结构化 JSON 仍作为内部表示（LLM 自然产出、未来交熔知入库用），但对外只交付报告，不另维护一套"双输出"。
7. **可独立运行，解耦上下游** —— 炼真是独立项目，可单独启动、有独立输出与监控目录，不依赖总管或上下游器；与上下游通过文件夹契约（中转文件）解耦，不 import 其他器代码。和准则 5「独立做、但对齐接口」一致——对齐的是文件格式，不是进程依赖。

## 六、做好了是什么样（验收标准）

1. **丢一条各平台（如 B站）的教程或经验文字进去，能拿到一份说人话的"鉴定报告"** —— 这条内容可信不可信、为什么、核心优点是什么、能照着做的步骤有哪些、来自哪个视频哪一段；报告还要**标出这条内容是否直接涉及变现（卖课/带货等）、以及它引用了哪些书或网址**。
2. **鉴定报告是给你判断要不要用，而不是一堆看不懂的分数** —— 结论能直接读，不用翻术语。
3. **可信的内容能顺畅交给熔知知识库存起来**（或先落本地、你能看到、能导出）。
4. **对卖课的虚假或夸大的内容，能标出"可疑 / 虚假"**，而不是照单全收。

## 七、和别的项目怎么配合（备注）

炼真只做中间"炼"这一步，上下游各有专人，互不抢活：

| 项目 | 角色 | 与炼真的关系 |
|---|---|---|
| 馏析 Nigredo | 采集 | 上游：把视频/文章变成文字交给炼真 |
| **炼真 Albedo** | **验真 + 提质** | **本项目** |
| 熔知 Citrinitas | 存储 | 下游：收炼真产出的干净内容入库（其底层的"分面分类"代码仍归它自己，不移交炼真） |
| 凝华 Rubedo | 变现 | 更下游：把可信 SOP 拿去用、去赚钱；也负责把成果封装成产品 |
| 总指挥部 OpusMagnum | 调度 | 未来负责意图理解（含"开天重写"伪前提校正） |

> 一句话边界：炼真**不采集、不存储、不变现、不重写你的意图、不封装产品**——这些分别是馏析 / 熔知 / 凝华 / 总指挥部 的活。炼真只把"验真 + 提质"这一件事做深做透。
